Desafio 01 - Perguntas:

  • 1. Para que serve o método Scrum?
    • A metodologia Scrum serve para o desenvolvimento de projetos complexos.

  • 2. Como funciona o método Scrum?
    • Para o funcionamento da metodologia Scrum deve-se seguir o seguinte ciclo: Planejamento, Implementação, Revisão e a Retrospectiva.

  • 3. O que é Git?
    • O Git é, basicamente, uma ordenação de comandoa que serve para monitorar algum desenvolvimento.

  • 4. O que é um scrum Product Owner?
    • O Product Owner, ou simplesmente P.O, é quem está diretamente ligado com o cliente e o time de desenvolvedores. Ele é responsável por definir as funcionalidades do produto de acordo com a necessidade do cliente.

  • 5. Qual o comando para criação de um novo repositório no Git?
    • Git init

  • 6. O que é o HTTP?
    • O Protocolo de Transferência de Hipertexto (HTTP) é um protocolo de transferência entre cliente-servidor que está localizado na camada de aplicação do TCP/IP.

  • 7. Como funciona o HTTP?
    • O HTTP é responsável por realizar a troca de dados na web (comunicação entre clientes e servidores por meio da internet) e permite a obtenção de recursos, como documentos HTML.

  • 8. Com o Git Você pode propor mudanças (adicioná-las ao Index) usando um comando. Qual é esse comando?
    • Git add . (para adicionar todos) ou Git add nome_do_arquivo (para adicionar um arquivo expecifíco).

  • 9. O que é a Branch master e para que serve?
    • Branch master é a parte principal do projeto.

  • 10. Quais são os comandos usados para atualizar um repositório local e fazer merge de um outro branch ao seu branch ativo?
    • Git pull

  • 11. Qual a diferença entre git e github?
    • O GitHub é uma plataforma que pode subir projetos e compartilha-los, já o Git é um software que serve para que os devenvolvedores possam desenvolver e atualizar os seus projetos para o uso do GitHub. Ou seja, eles trabalham em conjunto.

  • 12. Quais os dois verbos http que podemos utilizar para realizar um update? Explique a diferença entre eles.
    • PUT e PATCH, a principal diferença é que o PUT é utilizado para dados completos e o PATCH é para atualizações parciais.

  • 13. Qual o status code que pode ser usado na criação de um novo usuário?
    • 201 significa que a solicitação foi atendendida para a criação por meio do POST ou PUT.

  • 14. Quais são os três status code que podem ser utilizados para realizar o delete?
    • 200, 202 e 204.

  • 15. Exemplifique para que serve os metódos http 1xx, 2xx, 3xx, 4xx e 5xx. De uma forma macro (geral)!
    • 1xx(informativo) - significa que uma solicitação foi recebida.
    2xx(Sucesso) - significa que algo foi executado com sucesso.
    3xx(Redirecionamento) - significa que algo mudou e é necessário agir.
    4xx(Erro do cliente) - significa que o navegador enviou dados errados.
    5xx(Erro do servidor) - significa que o servidor apresenta algum problema.
